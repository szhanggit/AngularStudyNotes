	src\app\core\table.component.ts	(Due to the object is subject, it can be either Observer or Observable.)
	constructor(private model: Model, @Inject(SHARED_STATE) public observer: Observer<SharedState>) { }
	editProduct(key: number) {
        this.observer.next(new SharedState(MODES.EDIT, key));	//Emitter
    }

    createProduct() {
        this.observer.next(new SharedState(MODES.CREATE));		//Emitter
    }
	
	src\app\core\form.component.ts (Due to the object is subject, it can be either Observer or Observable.)
	constructor(private model: Model, @Inject(SHARED_STATE) public stateEvents: Observable<SharedState>) {
		stateEvents
		.subscribe(update => {									//The first handler for emitters above.
			this.product = new Product();
			if (update.id != undefined) {
				Object.assign(this.product, this.model.getProduct(update.id));
			}
			this.editing = update.mode == MODES.EDIT;     
		});
    }
	
	
	
	
Case 2: After clicking Edit button or Create button in Table form,
text will be displayed in the top banner.

1. The token is defined in src\app\core\sharedState.model.ts.
	export const SHARED_STATE = new InjectionToken("shared_state");
		
2. The valet is defined in src\app\core\core.module.ts
    providers: [{
        provide: SHARED_STATE,
        deps: [MessageService, Model],	//MessageService is a service type for injection. Model is a class type for injection.
        useFactory: (messageService, model) => { //messageService is an instantiated object of MessageService. model is an instantiated object of Model. 
            let subject = new Subject<SharedState>();
            subject.subscribe(m => messageService.reportMessage(	//m is an object of SharedState. Its content is defined at this.observer.next(new SharedState(MODES.EDIT, key));
                    new Message(MODES[m.mode] + (m.id != undefined
                        ? ` ${model.getProduct(m.id).name}` : ""))) //This is the second handler:
                );													//m => messageService.reportMessage(...)
            return subject;       
        }
    }]	
	
3. Will get a subject according to the SHARED_STATE token.
   The subject is registered the second handler in the provider.
