9. Create function annotation in Typescript
export interface UserInterface {
	id: string;
	name: string;
	age: number;
	getMessage(): string;
}

export interface ProfileInterface {
	name: string;
	profileUrl: string;
	isActive: boolean;
}


const transformUserToProfile = (user: UserInterface, isActive: boolean = true): ProfileInterface =>
{
	return {
		name: user.name,
		profileUrl: `/profiles/${user.name}`,
		isActive
	}
}

10. What is the purpose of tsconfig.json?
const optioins = {
	compilerOptions: {
		target: 'ES5',
		module: 'commonjs',
		moduleResolution: 'node',
		noImplicitAny: true,
	}
}



So we're using Tsconfig.json to configure TypeScript compiler, which actually means if you need to configure, for example, in what ECMAScript version, your TypeScript Transpiles JavaScript, this is exactly the place to change your configuration.
Target:	This is to what JavaScript version we will translate our typescript and here we have a configuration of ECMAScript five.
Module: want to use commonjs modules we have require and exports and inside ECMAScript six modules we have imports and exports.
moduleResolution: is a set in how our modules will be resolved.
noImplicitAny: Always need to specify type.

12. Why any is bad in Typescript?

let foo: any = 'foo';
console.log(foo.bar());

If use Any, typescript will automatically disable type checking.
Therefore, better not to use it.
If you have a function and you don't really know what will be the argument inside.
Solution 1: generic
Solution 2: Unknown
Neither of them will turn off type checking.

15. What are generics in Typescript?
const addId = <T>(obj: T) => {
	const id = Math.random().toString(16);
	return {
		...obj,
		id,
	};
};

const user = {
	name: 'Jack',
};

const result = addId<{name: string}>(user);
